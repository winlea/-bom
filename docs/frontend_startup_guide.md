# 前端服务启动指南

## 问题描述
前端服务无法启动，终端环境存在严重问题，命令被扭曲或无法正确执行。

## 解决方案

### 步骤1: 打开一个干净的终端窗口
1. 关闭所有现有的终端窗口
2. 打开一个新的、干净的终端窗口
3. 确保终端环境没有任何残留的命令或配置问题

### 步骤2: 启动前端服务
在干净的终端窗口中执行以下命令：

```bash
# 导航到前端项目目录
cd bom-redesign\bom-system-redesign

# 安装依赖（如果尚未安装）
npm install

# 启动开发服务器
npm run dev
```

### 步骤3: 验证服务启动
- 前端服务应该运行在 http://localhost:5174
- 后端服务运行在 http://localhost:5000

## 技术架构

### 后端
- **框架**: Flask + SQLite
- **服务地址**: http://localhost:5000
- **状态**: ✅ 运行中

### 前端
- **框架**: React + TypeScript
- **开发服务器**: Vite
- **默认地址**: http://localhost:5174
- **状态**: 需要在干净环境中启动

## 核心功能

- ✅ 项目管理 API
- ✅ BOM 管理 API
- ✅ 维度管理 API（符合ISO 1101-2004标准）
- ✅ 图像上传/下载 API
- ✅ 健康检查接口

## 注意事项

1. **终端环境问题**: 确保使用干净的终端窗口
2. **依赖安装**: 首次启动前需要运行 `npm install`
3. **端口冲突**: 如果端口 5174 被占用，Vite 会自动尝试其他端口
4. **CORS 配置**: 后端已配置支持前端跨域访问

## 故障排除

如果遇到以下问题：

1. **依赖安装失败**: 确保网络连接正常，尝试使用 `npm install --verbose` 获取详细错误信息

2. **端口被占用**: 关闭可能占用端口的其他服务

3. **编译错误**: 检查 `src` 目录下的 TypeScript 文件是否有语法错误

4. **API 连接问题**: 确保后端服务正在运行，且前端 API 配置正确指向 `http://localhost:5000`
