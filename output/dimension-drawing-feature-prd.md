# 尺寸绘制功能 PRD 文档

## 1. 功能概述

本功能旨在实现一个尺寸绘制系统，能够根据尺寸组合数量将画布分成相应部分，并在不同部分绘制不同类型的尺寸数据。具体来说，当有三个尺寸组合时，画布将被分为上中下三部分，分别绘制不同类型的尺寸数据。

## 2. 核心功能

### 2.1 画布分区功能
- 根据尺寸组合数量自动将画布分成相应部分
- 当有三个尺寸时，画布分为上中下三部分
- 当有两个尺寸时，画布分为两部分
- 分区大小均匀，确保各部分有足够空间绘制尺寸数据

### 2.2 尺寸绘制功能
- **上部区域**：绘制第一个尺寸数据
  - 如果是孔径尺寸，绘制孔径数据
  - 显示尺寸符号、名义值和公差
- **中部区域**：绘制第二个尺寸数据
  - 如果是空位置度，绘制基准框并填充数据
  - 显示位置度符号、公差值和基准
- **下部区域**：绘制第三个尺寸数据
  - 如果是空位置度，基准框与中部区域左边对齐
  - 生成可能只有一个基准的基准框
  - 填充尺寸数据

### 2.3 数据输入与配置
- 支持输入多个尺寸数据，包括尺寸类型、名义值、公差值和基准
- 支持配置画布大小、边距和间距
- 支持预览功能，实时查看绘制效果

### 2.4 导出与保存
- 支持导出绘制结果为图片格式
- 支持保存绘制配置和结果到数据库

## 3. 技术要求

### 3.1 前端技术
- 使用 React + TypeScript 实现
- 使用 Canvas API 进行绘制
- 兼容现代浏览器

### 3.2 后端技术
- 使用 Python + FastAPI 实现
- 支持保存绘制配置和结果到数据库

### 3.3 性能要求
- 绘制过程响应迅速，无明显延迟
- 支持处理至少 10 个尺寸的组合绘制

## 4. 实现方案

### 4.1 组件设计
- **DimensionDrawingCanvas**：核心绘制组件，负责根据尺寸数据绘制画布
- **DimensionDrawingConfig**：配置组件，负责接收用户输入的尺寸数据和配置参数
- **DimensionDrawingPreview**：预览组件，负责显示绘制结果

### 4.2 数据结构
```typescript
interface DimensionData {
  id: number;
  type: string; // 尺寸类型，如 'diameter'（孔径）、'position'（位置度）等
  nominalValue: string; // 名义值
  toleranceValue: string; // 公差值
  upperTolerance: string; // 上公差
  lowerTolerance: string; // 下公差
  datum: string; // 基准，如 'A', 'B', 'C' 等
}

interface DrawingConfig {
  canvasWidth: number;
  canvasHeight: number;
  padding: number;
  spacing: number;
  backgroundColor: string;
}
```

### 4.3 绘制流程
1. 接收用户输入的尺寸数据和配置参数
2. 根据尺寸数量计算画布分区
3. 遍历每个尺寸，根据类型和位置绘制相应的内容
4. 生成预览图片
5. 提供导出和保存功能

## 6. 风险与依赖

### 6.1 风险
- 尺寸数据输入错误可能导致绘制结果不正确
- 画布尺寸过小可能导致尺寸数据显示不全

### 6.2 依赖
- 依赖 Canvas API 进行绘制
- 依赖后端 API 进行数据保存
