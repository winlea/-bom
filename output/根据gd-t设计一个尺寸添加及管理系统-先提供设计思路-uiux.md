## 1. 设计概述

### 1.1 设计理念

- **简洁**: 去除不必要的元素
- **一致**: 统一的视觉语言
- **高效**: 快速完成任务
- **愉悦**: 细节打磨体验

### 1.2 设计原则

1. **用户中心**: 以用户需求为出发点
2. **数据驱动**: 基于数据迭代设计
3. **移动优先**: 响应式设计
4. **无障碍**: 符合 WCAG 2.1 AA

### 1.3 商业级 UI 执行红线（强制）

1. **禁止 AI 模板化视觉**：禁止直接输出同质化模板页面与缺少信息层级的“拼块式 UI”。
2. **禁止紫/粉渐变主视觉**：除非品牌规范明确要求，不允许使用紫色或粉色渐变作为主品牌视觉。
3. **禁止 emoji 充当功能图标**：功能图标统一使用专业 SVG 图标库（如 Lucide/Heroicons/Tabler）。
4. **禁止默认系统字体直出**：必须在文档中明确品牌字体组合与字号层级，页面必须体现明确字重与层级节奏。
5. **必须具备可访问交互**：可见 focus 态、可读对比度、键盘可达、并兼容 reduced-motion。
6. **必须有设计 token**：颜色、间距、圆角、阴影和动效时长需以 token 形式统一管理。

### 1.4 商业级体验目标

- UI 的首要目标不是“好看”，而是让用户快速理解产品价值、任务状态与下一步动作。
- 所有页面都应体现商业级完成度：品牌感、层级感、信息密度、状态完整度与信任表达。
- 当前项目应以“价值表达 + 核心流程可演示”为第一交付目标，并采用 medium 密度策略组织信息。
- 先解决页面结构、CTA、数据层级和组件一致性，再打磨视觉细节。
- 对外页面强调品牌、信任与转化，对内页面强调效率、清晰度和低认知负担。

### 1.5 设计 Intelligence 结论

- **界面定位**: 价值表达 + 核心流程可演示
- **信息密度**: medium
- **行业语气**: 现代、克制、有品牌感
- **首选组件生态**: shadcn/ui + Radix UI + Tailwind CSS
- **表单基线**: React Hook Form + Zod
- **数据展示基线**: Semantic table + lightweight sorting/filtering / Recharts

**优先原则**:
- 优先输出完整设计系统和页面骨架，而不是直接铺满页面装饰
- 页面需要像成熟商业产品，具备截图、案例、信任、状态、流程和数据层级
- 组件库只负责加速，不负责品牌感，必须做 token 和排版层重写
- 对营销页、产品页、工作台分别使用不同的信息密度和交互策略

**设计知识库关键词**:
- general / general / react / design tokens / component states / commercial ui / trust design

### 2.5 视觉方向与品牌感

- 视觉方向应基于产品定位建立明确气质：可信、克制、现代，而不是依赖泛滥渐变制造“高级感”。
- 优先使用有辨识度的字体组合、清晰的字号节奏、稳定的留白和克制的强调色。
- 图形、图标、插画、卡片阴影和分隔线应来自同一视觉系统，避免拼装感。
- 页面应在首屏就体现主价值、核心证据、下一步 CTA 和品牌记忆点。

### 2.6 布局栅格与密度策略

- 桌面端使用明确的 12 栏或等价栅格系统，控制内容宽度与节奏。
- 当前项目采用 medium 信息密度，按页面目标决定视觉节奏与操作密度。
- 控制不同页面的密度等级，避免同一产品里既过空又过挤。
- 侧边栏、头部、主体区、辅助区、底部应有稳定布局规则。

### 2.7 组件生态与实现基线

#### 首选方案: shadcn/ui + Radix UI + Tailwind CSS

**适用原因**: 适合需要现代品牌感、可定制组件和商业级页面控制力的 React/Next 项目。

**核心能力**:
- 组件骨架成熟，适合快速搭建高质量产品界面
- 与设计 token、主题系统、品牌定制兼容性强
- 便于结合 TanStack Table、React Hook Form、Zod 形成完整前端基线

**实现注意事项**:
- 必须先定义 token、容器、字重和信息密度，再批量生成组件
- 表单建议配合 React Hook Form + Zod

#### 配套技术基线

- **表单与验证**: React Hook Form + Zod
- **图表能力**: Recharts
- **表格/数据工作区**: Semantic table + lightweight sorting/filtering
- **图标体系**: Lucide + Simple Icons（仅品牌/合作方标识）
- **动效能力**: Framer Motion（营销/品牌页） + CSS transitions（工作台）

#### 可选备选方案

- **Ant Design**: 适合 B 端后台、运营系统和高密度表格工作台。
- **MUI**: 适合需要完整组件覆盖和企业级主题系统的 React 项目。

## 6. 交互设计

### 6.1 加载状态

- **页面加载**: 骨架屏
- **按钮加载**: Spinner + 禁用
- **数据加载**: Loading 动画

### 6.2 错误处理

- **网络错误**: 友好提示 + 重试按钮
- **表单错误**: 字段级提示
- **系统错误**: 错误页面 + 返回按钮

### 6.3 空状态

- **无数据**: 空状态插图 + 引导文案
- **搜索无结果**: 提示 + 清空按钮

### 6.4 动效与反馈系统

- 动效只服务于层级切换、状态反馈、焦点引导，不用作廉价装饰。
- 推荐实现基线: Framer Motion（营销/品牌页） + CSS transitions（工作台）。
- 列表进入、面板展开、Toast 反馈、模态切换应采用统一时长与缓动曲线。
- 对关键提交动作提供即时反馈，对长操作提供进度提示。
- 必须兼容 reduced-motion，必要时降级为无动画但保留状态反馈。

## 7. 响应式设计

### 7.1 断点定义

| 设备 | 宽度 | 说明 |
|:---|:---|:---|
| **Mobile** | < 640px | 手机竖屏 |
| **Tablet** | 640px - 1024px | 平板/手机横屏 |
| **Desktop** | 1024px - 1440px | 桌面 |
| **Large** | > 1440px | 大屏 |

### 7.2 响应式策略

- **Mobile First**: 从小屏开始设计
- **流式布局**: Flexbox + Grid
- **响应式图片**: srcset + sizes
- **响应式字体**: clamp() 函数

### 7.3 商业化与信任设计

- 对外页面应优先展示价值主张、能力边界、案例证明、客户证言、数据指标或合规信息。
- CTA 不应只在 hero 区出现，关键转化节点要有连续但不过度打扰的引导。
- 对内工作台则应突出当前任务、风险提示、审批状态、待办优先级和最近操作。
- 所有高价值页面都应体现“用户为什么相信并继续使用这个产品”。

**当前项目必须优先出现的信任/转化模块**:
- 能力边界说明
- 案例/截图/指标证明
- FAQ 与支持入口
- 状态反馈和错误恢复路径
- 价值说明 + 证据 + CTA

## 11. UX 最佳实践

基于项目特征，AI 为您推荐以下 UX 最佳实践：


### 1. Color (A11y)

**最佳实践**:
Use 4.5:1 contrast ratio for text

**避免**:
Light gray text on white background

**影响**: Readable by all users

**实施难度**: Low
